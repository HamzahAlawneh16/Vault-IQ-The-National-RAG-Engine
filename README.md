##  Project Overview
**Vault-IQ** is an enterprise-grade AI ecosystem built to handle sensitive national archives. It utilizes **Retrieval-Augmented Generation (RAG)** to provide accurate, context-aware answers from official documents. The system was designed to meet the rigorous standards of the **Ministry of Digital Economy and Entrepreneurship (MODEE)**, balancing high-performance AI with strict data privacy.

## 🧠 Technical Logic & Engineering Flow
I followed the **STAR Method** (Situation, Task, Action, Result) to document the core logic of this system:

### 1. The Ingestion Pipeline (The Factory)
* **Situation:** National documents are often large, unstructured PDFs containing sensitive info.
* **Task:** Extract, clean, and vectorize data without leaking private details.
* **Action:** * Integrated `PyMuPDF` for high-speed extraction.
    * Built a **Privacy Layer** using Regex engines to mask National IDs and Emails (Jordanian Standard).
    * Applied **Recursive Chunking** (500 chars window) to keep semantic context intact.
* **Result:** A secure, "clean" vector database ready for sub-second retrieval.

### 2. Semantic Retrieval (The Search)
* **Logic:** Instead of keyword matching, I used **Cosine Similarity** on 384-dimensional embeddings generated by `all-MiniLM-L6-v2`.
* **Result:** The system understands "Policy Intent." For example, searching for "Financial Support" will find "Salary Allowances" even if the exact words don't match.

## 🛠️ Detailed Project Roadmap (Step-by-Step)

### Phase 1: Modular Architecture
I decoupled the system into specialized modules: `core` for security, `db` for storage, and `services` for logic. This ensures that the system can be scaled in a cloud environment without crashing.

### Phase 2: The Security Firewall (PII Masking)
Before indexing, every sentence is scanned. If a National ID or Email is detected, it is replaced with `[MASKED]`. This ensures the AI learns from the "Logic" of the document, not the "Private Data" of citizens.

### Phase 3: RAG Orchestration
I connected the **FastAPI** backend to the **Qdrant Vector DB**. When a user asks a question, the system:
1.  Converts the question to a Vector.
2.  Fetches the top 3 most relevant "Chunks" from the vault.
3.  Synthesizes a final answer using an LLM prompt.
